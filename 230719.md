### [DB] DELETE vs TRUNCATE vs DROP

- DELETE: 데이터만 지워지고, 쓰고 있던 디스크 상의 공간 유지
- TRUNCATE: 최초 테이블이 만들어졌던 상태(= 데이터가 한 건도 없는 상태)로 돌아가며 컬럼 값만 남음
- DROP: 데이터와 테이블 전체를 삭제하여 모든 공간 반납



### [DB] 식별관계와 비식별관계

- 외래 키를 통해 테이블 간의 관계를 정립할 때 사용하는 전략
- 식별관계
  - 부모 테이블의 기본키 또는 유니크 키를 자식 테이블이 기본키로 사용하는 관계
  - 반드시 부모 테이블에 데이터가 존재해야 자식 테이블에 데이터를 입력할 수 있음
  - 부모 테이블에 자식 테이블이 종속
  - 장점
    - 데이터의 정합성 유지
    - 자식 테이블에 데이터가 존재한다면 부모 데이터도 반드시 존재함을 보장 가능
  - 단점
    - 요구사항 변경 시 구조 변경의 어려움
- 비식별관계
  - 부모 테이블의 기본키 또는 유니크 키를 자신의 기본키로 사용하지 않고 외래 키로 사용하는 관계
  - 자식 데이터는 부모 데이터가 없어도 독립적으로 생성될 수 있음
  - 자유로운 데이터 생성과 수정 가능
  - 장점
    - 변경되는 요구사항 수용 가능
    - 부모 데이터와 독립적인 자식 데이터 생성 가능
  - 단점
    - 데이터 정합성을 지키기 위해서는 별도의 비즈니스 로직 필요
    - 자식 데이터가 존재해도 부모 데이터가 존재하지 않을 가능성 있음 = 데이터 무결성 보장 X
